# 钱包超深度分析 Prompt v2

> **核心原则**：从"在哪里赚钱"出发，再看"怎么赚的"，最后还原"为什么这么做"。

---

## 分析逻辑演进

```
v1: 找到赚钱市场 → 看他怎么操作
v2: 找到赚钱市场 → 看他怎么操作 → 还原决策上下文 → 理解为什么能赚钱
```

**v2 核心增强**:
1. **更多市场覆盖**: TOP 10 盈利 + TOP 5 亏损 + 当前 TOP 5 持仓
2. **交易上下文还原**: 每笔交易时的市场状态
3. **已结算市场识别**: 区分"真正 Open" vs "已结算未赎回"

---

## 执行原则

### 原则 1: 问题驱动

```
想知道什么 → 找到答案在哪 → 定向获取 → 深度分析
```

### 原则 2: 漏斗式聚焦

```
宏观：账户概况
  ↓
中观：持仓分布（识别真正的 Open vs 已结算未赎回）
  ↓
微观：单个市场超深度分析
```

### 原则 3: 顺序执行，每步输出

```
完成一步 → 输出结果 → 再进入下一步
```

### 原则 4: 还原决策上下文

```
每笔交易，必须回答：
- 他在什么价格买/卖？
- 当时市场价格趋势是什么？（上涨/下跌/震荡）
- 他拿到的价格好不好？（vs 市场价）
- 这笔交易前后有什么重要事件？
```

## ⚠️ 关键数据清洗规则

### 已结算市场的识别

**问题背景**：
用户通常**不会 redeem 已输的市场**，所以当前持仓（Open Positions）中会混入：
1. ✅ 真正的 Open Positions（市场未结算）
2. ❌ 已结算但未赎回的亏损（显示为 `currentPrice = $0.00`, `percentPnl = -100%`）
3. ❌ 已结算但未赎回的盈利（显示为 `currentPrice = $1.00`, `percentPnl = +N%`）

**识别方法**：

```
Step 1: 观察 currentPrice
- currentPrice = 0.00 且 percentPnl = -100% → 高度疑似已结算亏损
- currentPrice = 1.00 且 percentPnl > 0 → 高度疑似已结算盈利未赎回

Step 2: 验证市场状态
调用 get_market(conditionId) 检查：
- 如果 market.closed = true 或 market.resolved = true → 确认是已结算市场
- 如果 market.acceptingOrders = false → 市场已关闭

Step 3: 分类处理
- 已结算亏损：从 Open Positions 移除，归入"历史亏损（未赎回）"
- 已结算盈利未赎回：提示用户可以 redeem
- 真正 Open：进行正常分析
```

**在报告中的处理**：

```markdown
### 2.1 当前持仓 (Open Positions)

⚠️ **数据清洗说明**：
原始持仓 X 个，其中：
- 真正的 Open Positions: Y 个
- 已结算亏损（未赎回）: Z 个 → 不计入"未实现亏损"
- 已结算盈利（未赎回）: W 个 → 建议尽快 redeem

以下分析仅针对真正的 Open Positions：
...
```

### Phase 4: 超深度市场分析

**关键增强**：每笔交易必须还原决策上下文

#### 对每个市场执行：

```
工具调用（顺序）：

Step 1: 获取市场信息
get_market(conditionId)
→ 市场标题、结果、状态、开始/结束时间

Step 2: 获取他在这个市场的所有操作
get_trader_activity(address, market=conditionId, limit=500)
→ 完整的 BUY/SELL/SPLIT/MERGE/REDEEM 序列

Step 3: 获取价格历史
get_klines(conditionId, interval='1h', outcome='both')
→ 价格走势 + Yes/No 双线 + 成交量

Step 4: 对比交易时间和价格走势
→ 判断入场时机的质量
```

> 卖出价格略优于市场价，执行质量良好。

| **最终收益** | $2,500 - 成本 = +$X |

> 在 $0.55 止盈 28% 后，他将剩余 72% 持有到结算。
> 这表明他对自己的判断有信心，愿意承担最后的不确定性换取完整收益。
