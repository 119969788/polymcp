# 聪明钱分析 Prompt v3

> **核心理念**：回答 3 个问题，支持 1 个决策
>
> 1. 他靠什么赚钱？（核心策略）
> 2. 为什么他能赚？（边缘来源）
> 3. 我能跟吗？（跟单价值）
>
> 决策：是否跟单 + 如何跟单

---

## v3 vs v2 对比

| 维度 | v2 | v3 |
|------|-----|-----|
| 分析市场数 | 20 个 | 5 个（最核心） |
| 策略数量 | 列举多个 | 聚焦 1 个核心 |
| 报告长度 | 500+ 行 | 200 行以内 |
| 输出重点 | 信息堆砌 | 决策支持 |
| 新增维度 | - | 跟单时效性、策略验证 |

---

## 分析框架

### Part 1: 快速画像（30 秒了解）

```
目标：一眼看懂这个人是谁

输出格式：

**一句话**：[类型] 交易者，靠 [核心策略] 在 [市场类型] 赚钱

| 指标 | 值 | vs 平均 |
|------|-----|---------|
| 总 PnL | $X | TOP X% |
| 胜率 | X% | [高于/低于] 平均 |
| 盈亏比 | X:1 | [高于/低于] 平均 |
| 活跃度 | X 天/笔 | [高频/低频] |
| 主战场 | [市场类型] | - |

**交易者类型**：
□ 冷门狙击手（低价高赔率）
□ 趋势跟随者（追涨杀跌）
□ 价值发现者（概率套利）
□ 事件驱动者（信息优势）
□ 做市商/套利 Bot

```

### Part 2: 核心策略识别（找到本质）

```
目标：找到他赚钱的 1 个核心策略，不是 10 个

分析方法：
1. 找到 TOP 3 盈利市场
2. 分析共性：入场价格、持仓周期、退出方式
3. 提炼核心策略

输出格式：

## 2. 核心策略

### 策略名称：[一个词概括，如"NHL 冷门狙击"]

### 策略公式

```
触发条件：
  市场类型 = [X]
  价格区间 = [< $X]
  其他条件 = [X]

入场方式：
  时机 = [X]
  仓位 = [总资金的 X%]
  执行 = [限价/市价]

退出方式：
  止盈 = [X]
  止损 = [X]
  默认 = [持有到结算]
```

### 证据（TOP 3 市场验证）

| 市场 | 入场价 | 退出价 | 盈亏 | 符合策略？ |
|------|--------|--------|------|-----------|
| A | $0.XX | $1.00 | +$X | ✅ |
| B | $0.XX | $1.00 | +$X | ✅ |
| C | $0.XX | $1.00 | +$X | ✅ |

### 策略一致性

- TOP 10 盈利市场中，X/10 符合此策略
- 策略覆盖了 X% 的总盈利
```

### Part 3: 边缘来源分析（为什么他能赚）

```
目标：找到他的 1 个主要边缘，不是 4 个都分析

四种边缘类型：
1. 信息边缘：比市场更早知道某些信息
2. 分析边缘：概率判断比市场更准
3. 执行边缘：交易执行比市场更好
4. 心理边缘：敢于逆向、坚持持有

分析方法：
- 信息边缘：入场时间 vs 关键事件时间
- 分析边缘：入场价格 vs 结算价格的平均差距
- 执行边缘：成交价 vs 市场价的平均差距
- 心理边缘：低价重仓次数、持有到结算比例

输出格式：

## 3. 边缘来源

### 主要边缘：[X 边缘]

**证据**：
- [具体数据支撑]
- [具体数据支撑]
- [具体数据支撑]

**量化**：
- 边缘强度 = X%（他 vs 市场平均）

### 边缘来源矩阵

| 边缘类型 | 强度 | 证据 |
|----------|------|------|
| 信息边缘 | ⬜⬜⬜⬜⬜ | 无明显证据 |
| 分析边缘 | ⬛⬛⬛⬜⬜ | 入场价平均低于结算价 X% |
| 执行边缘 | ⬛⬛⬛⬛⬜ | 成交价优于市场价 $0.0X |
| 心理边缘 | ⬛⬛⬜⬜⬜ | X% 持有到结算 |

**结论**：他的核心竞争力是 [X]，普通人 [能/难以] 复制
```

### Part 4: 跟单价值评估（能不能跟）

```
目标：评估跟单的可行性和预期收益

这是 v3 新增的核心部分！

分析维度：
1. 信号时效性：他入场后，价格涨多少？
2. 跟单窗口：最佳跟单时机是多久？
3. 跟单成本：延迟跟单损失多少？
4. 跟单风险：他亏钱时的模式？

输出格式：

## 4. 跟单价值评估

### 4.1 信号时效性

分析他的 TOP 10 盈利市场：

| 市场 | 入场价 | T+1h | T+6h | T+24h | 结算价 |
|------|--------|------|------|-------|--------|
| A | $0.30 | $0.32 | $0.38 | $0.45 | $1.00 |
| B | $0.25 | $0.26 | $0.30 | $0.35 | $1.00 |
| ... | ... | ... | ... | ... | ... |
| **平均** | $0.XX | +X% | +X% | +X% | +X% |

**结论**：
- 入场后 1h 平均涨幅：X%
- 入场后 6h 平均涨幅：X%
- 入场后 24h 平均涨幅：X%

### 4.2 最佳跟单窗口

```
如果在他入场后立即跟单：预期收益 X%
如果在他入场后 1h 跟单：预期收益 X%（损失 X%）
如果在他入场后 6h 跟单：预期收益 X%（损失 X%）
如果在他入场后 24h 跟单：预期收益 X%（损失 X%）

最佳跟单窗口：[X 小时内]
```

### 4.3 跟单成本分析

| 成本类型 | 估算 |
|----------|------|
| 延迟成本 | 每延迟 1h 损失 X% |
| 滑点成本 | 大约 X% |
| 信息获取成本 | [实时监控需要 X] |

### 4.4 跟单风险分析

分析他的 TOP 5 亏损市场：

| 市场 | 入场价 | 结算价 | 亏损 | 失败原因 |
|------|--------|--------|------|----------|
| X | $0.XX | $0 | -$X | [原因] |

**亏损模式**：
- 他在 [X 类型] 市场容易亏损
- 亏损时的共同特征：[X]

**跟单避坑规则**：
- 当 [条件] 时，不跟单
- 当 [条件] 时，减小仓位

### 4.5 跟单价值评分

| 维度 | 评分 | 说明 |
|------|------|------|
| 信号质量 | ⭐⭐⭐⭐⭐ | 胜率 X%，盈亏比 X:1 |
| 时效性 | ⭐⭐⭐⭐ | 24h 内跟单仍有 X% 收益 |
| 可监控性 | ⭐⭐⭐ | 需要 [X] 监控 |
| 可执行性 | ⭐⭐⭐⭐ | 单笔 $X 可复制 |

**总体跟单价值**：[高/中/低]
```

```
输出格式：
```

## 执行流程

```
Phase 1: 数据获取（5 分钟）
├── get_trader_profile
├── get_account_value
├── get_trader_closed_positions (TOP 50 盈利)
├── get_trader_closed_positions (TOP 50 亏损)
└── get_trader_positions (当前持仓)

Phase 2: 快速画像（2 分钟）
└── 输出 Part 1

Phase 3: 核心策略识别（10 分钟）
├── 分析 TOP 3 盈利市场
├── 提炼核心策略
└── 输出 Part 2

Phase 4: 边缘来源分析（5 分钟）
├── 计算四种边缘的证据
├── 确定主要边缘
└── 输出 Part 3

Phase 5: 跟单价值评估（10 分钟）⭐ 新增
├── 分析信号时效性
├── 计算最佳跟单窗口
├── 分析亏损模式
└── 输出 Part 4

Phase 6: 策略验证（5 分钟）⭐ 新增
├── 回测策略表现
└── 输出 Part 5

Phase 7: 一页纸总结（2 分钟）
└── 输出 Part 6
```

## v3 核心改进总结

| 改进点 | v2 | v3 |
|--------|-----|-----|
| 分析目标 | 尽可能多的信息 | 回答 3 个核心问题 |
| 策略数量 | 列举多个 | 聚焦 1 个核心 |
| 边缘分析 | 四个都分析 | 找到主要 1 个 |
| **跟单评估** | 无 | ⭐ 新增信号时效性 |
| **策略验证** | 无 | ⭐ 新增回测 |
| 报告长度 | 500+ 行 | 200 行以内 |
| 输出重点 | 考古报告 | 决策支持 |

**v3 的本质**：从"告诉你他做了什么"变成"告诉你该不该跟、怎么跟"
