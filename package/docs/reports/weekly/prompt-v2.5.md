# 聪明钱超深度考古 Prompt v2.5

> **核心理念**：像侦探一样还原每一个交易决策的全过程
>
> 不只是"他做了什么"，而是：
> - 他在什么情况下做的？
> - 他是怎么执行的？
> - 市场当时是什么状态？
> - 他的决策逻辑可能是什么？

---

## ⚠️ API 数据能力与限制

**在开始分析前，必须了解各 API 的数据能力：**

### 数据源对比

| 数据源 | API | 历史深度 | 关键限制 |
|--------|-----|----------|----------|
| **活动历史** | `get_trader_activity` | ⚠️ 仅近期 (~2h) | Activity API 不保留历史，只有近期数据 |
| **已平仓位** | `get_trader_closed_positions` | ✅ 完整历史 | 可获取所有已结算的持仓和 PnL |
| **当前持仓** | `get_trader_positions` | ✅ 当前快照 | 包含已结算未赎回的持仓 |
| **市场交易** | `get_market_trades` | ⚠️ 近期 | 特定市场的近期交易记录 |
| **K线数据** | `get_klines` | ⚠️ 有限 | 取决于市场活跃度 |
| **Orderbook** | `get_orderbook` | ✅ 实时 | 只有当前快照，无历史 |

### 分析策略调整

由于 Activity API 历史数据有限，v2.5 的深度分析应：

1. **优先使用 Closed Positions** - 完整的历史盈亏数据
2. **结合 Market Trades** - 对重点市场获取交易记录
3. **近期活动** - 用于识别当前交易模式和 Bot 特征
4. **K线辅助** - 还原市场价格走势

### 可实现 vs 理想化

| 分析维度 | 可实现程度 | 依赖数据 |
|----------|-----------|----------|
| 盈亏来源 | ✅ 完全可实现 | Closed Positions |
| 持仓分布 | ✅ 完全可实现 | Positions |
| Bot 检测 | ⚠️ 仅限近期 | Activity (近期) |
| 执行质量 | ⚠️ 仅限近期 | Activity, Market Trades |
| 时间模式 | ⚠️ 仅限近期 | Activity (近期) |
| 仓位演变 | ❌ 无法还原 | 需要完整 Activity 历史 |
| 交易时间线 | ⚠️ 部分 | Market Trades (特定市场) |
