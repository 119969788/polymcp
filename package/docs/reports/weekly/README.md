## 研究过程中遇到的问题

### 1. API 限制问题

| 问题 | 描述 | 解决方案 |
|------|------|---------|
| 活动数据过期 | `get_trader_activity` 无法获取超过一定时间的历史活动，部分市场的具体交易时间线无法还原 | 使用 `get_trader_closed_positions` 获取已平仓数据作为替代，但缺少具体时间点 |
| 分页限制 | `get_trader_closed_positions` 最多返回 50 条，`get_trader_positions` 最多返回 500 条 | 使用 offset 参数分页获取，但增加了 API 调用次数 |
| 排序参数不一致 | 不同接口的 sortBy 参数支持的值不同 | 需要针对每个接口使用正确的参数 (如 CASHPNL vs REALIZEDPNL) |

### 2. 数据质量问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 已结算持仓显示 $0 | 当前持仓中包含已结算但未赎回的市场，价格显示为 $0.00 | 误导性的"亏损"显示，实际可能是盈利但未赎回 |
| 胜率计算偏差 | Profile 中的胜率基于所有持仓，包含未结算的亏损持仓 | 已平仓胜率 (如 Ems123 87.9%) 与总胜率 (38%) 差异巨大 |
| Bot 交易识别 | SPLIT/MERGE 操作频繁的账户可能是套利 Bot，但无法确认 | 部分"聪明钱"可能是算法交易而非人工判断 |

### 3. 分析方法问题

| 问题 | 描述 | 改进建议 |
|------|------|---------|
| 边缘来源难以验证 | 无法确定交易者的信息来源（是否有内幕、专业分析团队等） | 只能通过交易模式推测，建议跟踪长期表现 |
| 单周数据偏差 | 本周表现好不代表长期稳定盈利 | 需要更长周期的数据分析 |
| 体育知识门槛 | 大部分盈利来自体育博彩，需要专业知识才能复制 | 需要体育领域专家参与分析 |

## 可复制策略总结

### 高可复制性策略

1. **Spread 反热门策略** (gmanas, Latina, gmpm)
   - 条件: 点差市场，热门让分 > 6.5
   - 执行: 押注劣势方 (被让分方)
   - 原理: 市场倾向高估热门队伍的让分能力
   - 门槛: 中等资金 ($10K+)，基础体育知识

2. **NHL Underdog 策略** (kch123, 0x2E98)
   - 条件: NHL 比赛，客队价格 < $0.40
   - 执行: 押注 Underdog
   - 原理: NHL 冷门频率高于其他联赛
   - 门槛: 低-中等资金，NHL 基础知识

### 中等可复制性策略

1. **赛中加仓策略** (Ems123, kch123)
   - 条件: 已有赛前持仓，比赛进行中形势有利
   - 执行: 使用 API/Bot 快速加仓
   - 门槛: 需要交易 Bot + 实时观看比赛

2. **不对称对冲策略** (gmpm, Ems123)
   - 条件: 对比赛有倾向但不确定
   - 执行: 双向押注，倾向方押更多
   - 门槛: 理解对冲逻辑，较高资金

### 低可复制性策略（需特殊资源）

1. **大单冷门狙击** (bossoskil)
   - 需要: $200K+ 资金，专业体育分析能力
   - 风险: 单笔亏损可能超过 $100K
   - 门槛: 极高，不建议普通用户尝试

2. **高频 Live Betting** (kch123)
   - 需要: 自动化交易系统，实时数据源
   - 风险: 技术故障、延迟导致错误交易
   - 门槛: 需要开发/技术能力

## 核心结论

### 本周聪明钱的共同特征

1. **逆市场思维**: 敢于在市场共识的反方向下注
2. **体育专精**: 90% 盈利来自体育博彩，政治/加密市场罕见
3. **大单策略**: 发现机会后果断重仓，而非分散小注
4. **技术手段**: 广泛使用交易 Bot 提高执行效率
5. **风险承受**: 能承受大额单笔亏损，不被短期波动影响

### 普通用户的建议

| 建议 | 说明 |
|------|------|
| **不要盲目跟单** | 聪明钱的边缘来自专业知识和大资金，普通用户难以复制 |
| **从 Spread 开始** | Spread 反热门策略相对简单，可以小资金尝试 |
| **专注单一领域** | 选择你熟悉的联赛 (如 NBA)，深入研究而非广泛参与 |
| **控制仓位** | 单笔不超过总资金 10%，保持长期生存能力 |
| **使用 API** | 如果可能，使用 API 交易提高执行效率 |

## 分析方法论

使用的分析框架：[prompt.md](./prompt.md)

核心步骤：
1. 账户概况 (Profile + Account Value)
2. 持仓分析 (Open + Closed Positions，按 PnL 排序)
3. TOP 市场深度分析 (盈利来源 + 亏损案例)
4. 行为画像 (交易风格、时间规律、技术手段)
5. 边缘来源分析 (信息优势、执行优势、资金优势)
6. 策略提取 (可复制的交易逻辑)
7. 风险分析 (已暴露风险 + 潜在风险)
